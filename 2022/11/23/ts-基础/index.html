ts与js的区别

1、ts 可以在编译过程中发现错误，而 js 只能在运行阶段发现错误
2、ts 类型为强类型、静态、动态类型、js 为弱类型
3、ts 最终会编译成js, js 则直接可以在浏览器中运行
4、ts 支持泛型、模块、接口

①、ts 中的基础类型

Boolean 、Number 、String 、Symbol 、Array 、Enum
数组为两种展现形式
数字枚举 初始值为0，当然也可以设置初始值
如下为提供的案例

Any、Unknown、never

在 ts 中 任何类型都可以为any类型，它是 ts 中的一个 “逃逸舱” 至此any也被称为顶级类型
所有的类型都可以赋值给any,所有类型也都可以赋值给 unknown
unknown 类型 只能被赋值给 any 类型和 unknown 类型本身
never类型 表示的是那些永不存在的值的类型。
没有类型是 never 的子类型，没有类型可以赋值给 never 类型（除了 never 本身之外）。 即使 any也不可以赋值给 never 。

Tuple 类型

undefined 、 null

默认情况下 null 和 undefined 是所有类型的子类型

TypeScript 断言

1、类型断言

1、类型断言有两种形式：“尖括号” 语法、as 语法

2、非空断言

在上下文中当类型检查器无法断定类型时，一个新的后缀表达式操作符 ! 可以用于断言操作对象是非 null 和非 undefined 类型
x! 将从 x 值域中排除 null 和 undefined

3、确定赋值断言

1、允许在实例属性和变量声明后面放置一个 ! 号，从而告诉 TypeScript 该属性会被明确地赋值。

类型守卫

类型保护是可执行运行时检查的一种表达式，用于确保该类型在一定的范围内。
in、typeof、instanceof、自定义类型保护的类型谓词
typeof 类型保护只支持两种形式：typeof v === “typename” 和 typeof v !== typename
“typename” 必须是 *”number”*， *”string”*， “boolean” 或 “symbol”。 但是 TypeScript 并不会阻止你与其它字符串比较，语言不会把那些表达式识别为类型保护

interface 和 type的区别

interface（接口）和 type（类型别名）的对比：

相同点：都可以给对象指定类型

不同点:
interface ：
只能为对象指定类型
可以使用extends继承
多个同名的interface会合并

type：
不仅可以为对象指定类型，实际上可以为任意类型指定别名
可以使用&运算符实现继承效果
多个同名的type会报错

TypeScript 泛型

泛型

以下是常见泛型变量代表的意思：
K (key) 表示对象中的键类型
V (value) 表示对象中的值类型
E (Element) 表示元素类型

当然类型变量可以为多个

泛型

泛型接口

在接口名称的后面添加 *<类型变量>*，那么，这个接口就变成了泛型接口。
接口的类型变量，对接口中所有其他成员可见，也就是接口中所有成员都可以使用类型变量。
使用泛型接口时，需要显式指定具体的类型 (比如，此处的 IdFunc)。
函数返回时：id 方法的参数和返回值类型都是 number;ids 方法的返回值类型是 number[]。
